# 钉钉组织与系统权限映射

钉钉只提供员工身份、部门和角色；本系统在服务端把它们映射为业务门店、业务角色和默认入口。浏览器不直接根据部门名称决定权限。

## MVP1 自动规则

MVP1 不要求管理员在 H5 中配置角色 ID。后端根据钉钉角色名称精确匹配：

- `服务顾问` → `advisor` → `order-create`
- `门店管理员` → `manager` → `workbench`

员工只需加入钉钉组织，并由管理员分配上述角色。首次从钉钉工作台进入 H5 时，系统会自动创建账号并默认归属 `shop-hq`（抚顺路店）。职位、岗位和备注中的同名文字不参与权限判断，必须是钉钉角色。

数据库映射表继续保留为多门店扩展能力。将来需要不同名称或不同门店时，显式角色 ID/部门 ID 映射会优先于内置名称规则。

## 登录行为

每次钉钉免登成功后，后端会：

1. 通过 `authCode` 获取钉钉 `userid`。
2. 获取员工详情并保存部门/角色快照。
3. 优先匹配显式映射；没有显式映射时使用 MVP1 内置角色名称规则。
4. 自动创建或更新 `users` 中的姓名、手机号、角色、门店和启用状态。

无论员工是否曾经手动绑定，只要当前没有“服务顾问”或“门店管理员”钉钉角色，系统都会拒绝登录并提示联系钉钉管理员分配角色。

## 钉钉后台权限

应用需具备获取当前员工身份和读取员工基础详情（含部门、角色）的权限；权限变更后需要在钉钉后台重新发布应用。员工调岗、角色调整会在下次登录时同步。实时事件订阅可以在后续版本加入，用于无需登录的即时同步。

## 安全边界

- `DINGTALK_APP_SECRET` 和 `JWT_SECRET` 只能放在服务器 `.env.production`。
- 只有系统 `manager` 能读写 `/api/admin/dingtalk-*` 配置接口。
- 不要在 Vite 前端环境变量或浏览器代码中放钉钉密钥。
